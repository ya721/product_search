## MCP-Server-Git 自動使用指示

**自動実行原則**
- 全てのGit関連作業においてmcp-server-gitツールを必ず使用する
- リポジトリの状態確認、ブランチ操作、コミット処理を自動化する
- 手動でのGitコマンド実行は禁止し、必ずMCPツールを経由する
- エラーや異常が検出された場合は即座に報告し、自動修正を試行する
- 開発は常に`develop`ブランチを基点とし、`main`ブランチには直接マージしない
- リアルタイムでGit状態を監視し、ファイルに変更があった場合は自動コミットを実行する

**必須自動実行タスク**
1. **リポジトリ状態の常時監視**: 作業開始時と各操作後にリポジトリ状況を自動確認
2. **ブランチ戦略の自動検証**: 命名規則、親ブランチ関係、競合チェックを自動実行
3. **コミット品質の事前検証**: 単一責任原則、メッセージ形式、影響範囲を自動分析
4. **自動化レポート生成**: 全ての操作後に状況レポートを自動生成

## 基本原則

- mcp-server-git を活用した自動化されたGit操作
- 型に応じたブランチ戦略(feature、fix、refactor等の明確な命名規則)
- 最小単位コミット(単一責任の原則に基づく原子的な変更)
- リアルタイムでのGit状態監視と検証
- "修正" "更新" "変更" などの曖昧な表現は使用しない
