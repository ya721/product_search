# 承認要求の検証手順

管理者は、以下の手順に厳格に従って承認要求を検証してください。

1. **反証（実証的証拠に基づく）**
  * 承認要求内容に対して、反証可能な点を見出し、実証的な証拠を用いて反証してください。
  * **あなたの反証内容をClaudeで確認してください**: `claude -p "反証内容の確認"`
  * 管理者とClaudeの反証結果を踏まえて、反証に対する心証（信頼性、妥当性、説得力など）を述べてください。
2. **正当性の立証（実証的証拠に基づく）**
  * 承認要求内容を具体的な実証的証拠を提示して、承認要求内容の正当性を立証してください。
  * **あなたの正当性を立証してClaudeで確認してください**: `claude -p "正当性の確認"`
  * 管理者とClaudeの正当性の立証結果を踏まえて、正当性の立証に対する心証（信頼性、妥当性、説得力など）を述べてください。
3. **心証の提示**
  * 正当性の立証および反証の結果を踏まえ、あなたの心証（信頼性、妥当性、説得力など）を述べてください。
  * **あなたと心証内容をClaudeで確認してください**: `claude -p "心証内容の確認"`
  * 管理者とClaudeの心証結果を踏まえて、心証に対する最終的な判断を述べてください。

```markdown
# 要求内容の検証結果

## 反証内容
- [反証内容の要約]

## 正当性の立証
- [正当性を要約]

## 心証の提示
- [心証の要約]

## 結果
- [DISAPPROVAL/APPROVAL/CONDITIONAL_APPROVAL]

```
